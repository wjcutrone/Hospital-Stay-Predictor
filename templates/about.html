This prediction model was made using a data set found on kaggle.com, originally from a Analytics Vidhya Hackathon on Healthcare Analytics. After cleaning the data in Google Colaboratory, building the model with Tensorflow, and fitting the model with the data, it was able to predict the length of a hospital stay with about 42% accuracy. The main limitations were the size of the data set as well as little to no documentation on the meaning of the values. Nevertheless, if a user hypothetically had access to all of the information required, they could use this model to make a predication. This could be useful for a patient's own knowledge and efficiency in selecting a hospital, or for hospital staff to keep track of and make decisions for their patients.


Key:

Hospital Code: Unique identifier for the hospital
Hospital Type Code: Unique identifier for the type of hospital
Hospital City Code: City code for the hospital
Region Code: Region code for the hospital
Extra Rooms Available: Number of extra rooms available
Department: Department handling the patient
Ward Type: Code for the type of ward
Ward Facility Code: Code for the facility of the ward
Bed Grade: Condition of Bed
Patient City Code: City code for the patient
Type of Admission: Description of patient's admission
Severity of Illness: Severity of Illness at time of admission
Visitors with Patient: Number of visitors patient has
Age: Age of patient
Admission Deposit: Amount deposited for stay at time of admission
